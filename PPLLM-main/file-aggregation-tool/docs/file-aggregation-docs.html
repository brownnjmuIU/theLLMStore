File Aggregation Tool

Prepare your files for LLM processing, RAG, and document Q&A

What It Does

The File Aggregation Tool crawls your home directory, identifies file types, and generates a structured JSON catalog of your documents, code, images, and data. It helps you decide which files to feed into LLMs for RAG (Retrieval-Augmented Generation), document Q&A, fine-tuning, or local chat—before you upload or process anything.

High-Level Workflow

Scan → Analyze → Review by category → Select files → Export JSON

The app scans your user folder (Documents, Desktop, Downloads, etc.) while excluding system and cache directories. Each file is classified (document, code, image, data, archive) and given a one-line summary. You pick only the files you want, then export them as JSON—ready for downstream LLM pipelines.

Features

  • Automatic discovery — Starts from your home directory; no path selection upfront
  • File type detection — Documents (PDF, Word, Excel), code (Python, JS, etc.), images, video, audio, data (JSON, CSV), archives
  • One-line summaries — Content previews or metadata (e.g., image dimensions, JSON structure, code line counts)
  • Category-based UI — Browse files grouped by type with checkboxes to select
  • Selective export — Export only the files you choose as JSON
  • Standalone .exe — Can be packaged for Windows (or native binary on macOS/Linux) so users run it without Python

Use Cases for theLLMStore

Use the File Aggregation Tool to curate which documents and code you feed into:

  • RAG pipelines — Select relevant docs, export JSON, then index for retrieval
  • Document Q&A — Identify PDFs, Word files, and text docs before chunking and embedding
  • Fine-tuning — Find and export training data (JSON, CSV, code) for LoRA or full model training
  • Local chat / OpenClaw — Know what files you have before connecting them to your LLM

Quick Start

Run the GUI with python3 gui_app.py. Click Scan & Aggregate, review files by category, select the ones you want, then click Export Selected to JSON. The JSON includes paths, types, sizes, summaries, and metadata—ready for your LLM workflow.
